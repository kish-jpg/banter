<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| device → backend (network request DTOs) | AnalyzeConversationRequest is the only shape that would cross to the backend; it must never carry raw image bytes (CAPT-04). No live network call exists this phase — the boundary is a compile-time type + build-time test guarantee. |
| app process ↔ keyboard process (App Group container) | Data written by the app is read by the keyboard via UserDefaults(suiteName:); both must use the identical AppGroupStore.suiteName. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-04 | Information Disclosure | AnalyzeConversationRequest (NetworkDTOs.swift) | high | mitigate | Request type declares only String/primitive/model fields; NetworkBoundaryGuardTests fails the build if UIImage/: Data/[UInt8]/CGImage appears in the DTO source. Structural enforcement of CAPT-04. |
| T-01-05 | Tampering | AppGroupStore.suiteName drift between targets | medium | mitigate | suiteName declared exactly once (grep asserts count == 1); both targets reference AppGroupStore.suiteName, never a duplicated literal — prevents silent app/keyboard container mismatch. |
| T-01-06 | Information Disclosure | App Group container reused by an unintended target | low | accept | suiteName `group.com.banter.shared` is uniquely namespaced under this project's bundle prefix; entitlement wiring verified in plan 03. No sensitive real-user data in the skeleton's sample payloads. |
</threat_model>
