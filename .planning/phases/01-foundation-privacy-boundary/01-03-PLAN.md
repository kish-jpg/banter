<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app process ↔ keyboard process | Both targets access the same App Group container; a mismatched App Group ID between their entitlements would silently break the handoff. |
| keyboard extension → host app / system | The keyboard runs sandboxed with RequestsOpenAccess NO this phase — no network, read-only App Group access. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-07 | Tampering | App Group ID mismatch across entitlements | high | mitigate | Both entitlements files and project.yml carry the byte-identical `group.com.banter.shared`; verify step greps all three. Matches AppGroupStore.suiteName (plan 02). |
| T-01-08 | Information Disclosure | keyboard requesting Full Access unnecessarily | medium | mitigate | RequestsOpenAccess NO in the keyboard Info.plist for Phase 1 — the skeleton needs only read access to the App Group, not network. Avoids premature Full-Access scrutiny (STACK.md). |
| T-01-09 | Elevation of Privilege | duplicate model definitions drifting from BanterShared | medium | mitigate | Grep asserts neither target redeclares ConversationMessage/ReplySuggestion/SentimentEvent; types resolve only from BanterShared (single source of truth). |
</threat_model>
