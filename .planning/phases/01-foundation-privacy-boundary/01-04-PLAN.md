<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pushed source → GitHub Actions runner | The workflow executes repo-authored YAML/Swift on a hosted macOS runner; only trusted first-party/well-known actions are used. |
| generated project → build assertions | XcodeGen may silently drop a key (schema drift); the entitlement-grep assertion is the trust check that generation actually produced the App Group wiring. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-10 | Tampering | untrusted GitHub Actions in the workflow | high | mitigate | Use only first-party/well-known actions (actions/checkout, actions/cache, maxim-lobanov/setup-xcode) verified in RESEARCH Package Legitimacy Audit; no unvetted marketplace actions. |
| T-01-11 | Spoofing / silent failure | XcodeGen schema drift dropping the App Group entitlement | medium | mitigate | Explicit grep assertion on the generated entitlements fails the job if `group.com.banter.shared` is missing — never trust the xcodegen exit code alone (RESEARCH Pitfall 2). |
| T-01-12 | Information Disclosure | CI signing material / secrets | low | accept | No signing certs or App Store Connect keys exist this phase (simulator-only, no signing); when Phase 5/8 add a paid account, use GitHub encrypted secrets and never commit certs (RESEARCH Security Domain). |
</threat_model>
