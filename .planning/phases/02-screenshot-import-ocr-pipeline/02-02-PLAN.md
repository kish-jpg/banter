<objective>
Turn raw OCR lines into a correctly-ordered, speaker-attributed `[ConversationMessage]` — the deterministic geometry heart of CAPT-01. This is where the two documented hard pitfalls live: Vision's bottom-left bounding-box origin (reading order silently reverses if mishandled) and the x-alignment attribution threshold. Both get explicit unit tests against fixtures.

Purpose: Produce the transcript the confirm screen (plan 04) reviews. Deterministic, testable without a simulator boot for the logic — though the fixture→OCR→attribution end-to-end still runs in CI.
Output: `BubbleAttributor.attribute(_:)`, a documented `0.4` x-threshold constant with a `ponytail:` tuning comment, a noise-filter, and fixture tests covering reading order + attribution + noise.
</objective>
