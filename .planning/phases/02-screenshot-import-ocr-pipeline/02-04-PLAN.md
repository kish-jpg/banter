<objective>
Build the three-screen SwiftUI flow that is the user-visible product of this phase: Import Entry → Parsing Progress → Confirm Transcript. This is the MVP vertical slice — after it a real user can go from a screenshot (or pasted text) to a confirmed, corrected transcript. It also establishes Banter's FIRST design-token system (per 02-UI-SPEC.md), reused unchanged in Phase 4.

Purpose: CAPT-02's confirm/flip/edit loop is the correctness net for the probabilistic OCR heuristic — the whole reason attribution is allowed to be approximate. This screen is where the user exercises the CAPT-04 gate.
Output: `BanterTokens.swift` (Colors/Spacing/Type/Radius), color assets, `ImportEntryView` (PhotosPicker + paste), `ParsingProgressView`, `ConfirmTranscriptView` (flip/edit/confirm), `ImportFlowModel` (state machine wiring OCRPipeline/BubbleAttributor/PasteTextParser), and a real entry point replacing the Phase-1 sample writer.
</objective>
