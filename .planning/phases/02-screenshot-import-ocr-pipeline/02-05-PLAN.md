<objective>
Deliver the USER-REQUESTED capability: CI produces viewable app screenshots as workflow artifacts on every run. With no Mac, this is the developer's only channel to visually verify the Phase-2 UI. Uses the Apple-blessed XCUITest + XCTAttachment(.keepAlways) + actions/upload-artifact@v4 pattern (RESEARCH), and — unlike Phase 1's SPM-test-into-scheme problem — an XCUITest target IS a native Xcode construct XcodeGen supports attaching to a scheme's test action.

Purpose: Close the phase's verification loop. Kish reviews the uploaded screenshots to confirm the confirm-screen UI looks right (and later to eyeball attribution against real screenshots). This is best-effort visual coverage of CAPT-02's confirm UI.
Output: `BanterUITests` XCUITest target, `ScreenshotArtifactTests`, project.yml target wiring, and CI steps running the UI test + uploading the `.xcresult`.
</objective>
