<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| edge function → Gemini API | Outbound call carrying GEMINI_API_KEY; the key must never appear in a response body or log. |
| transcript text → prompt | User-supplied transcript is interpolated into the prompt; a crafted message could attempt to override the system instruction (prompt injection). |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-04 | Information Disclosure | GEMINI_API_KEY leaking into logs or the response | high | mitigate | Adapter reads the key from a constructor arg (sourced from Deno.env in 03-03), never logs it, never returns it. GeminiAdapter.test.ts asserts the key is only in the outbound request. Client-side key tripwire is added in 03-03. |
| T-03-05 | Tampering / Elevation of Privilege | Prompt injection via transcript text to bypass the allowlist | high | mitigate | The prompt only injects allowlisted vocabulary; the real defense is the 03-01 validator applied in 03-03 after generation. This plan's prompt is defense-in-depth, not the gate (RESEARCH Anti-Patterns). |
| T-03-06 | Denial of Service (cost) | Oversized transcript inflating Gemini token spend | medium | mitigate | Request-length caps are enforced in index.ts request validation (03-03, V5); this plan keeps formatTranscript pure so the cap can wrap it. Noted as a cross-plan handoff. |
</threat_model>
