## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → coaching function | POST body (transcript/profile text) crosses from the untrusted iOS app into the backend; must be validated (non-empty, enum tone, length cap). |
| edge function env → repo/client | GEMINI_API_KEY lives only in the edge function env; it must never reach client source or CI logs. |
| tracked Backend/ → gitignored volume | The sync step copies code into an untracked dir; the tracked copy is the source of truth. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-04 | Information Disclosure | GEMINI_API_KEY in client source or CI logs | high | mitigate | GeminiKeyBoundaryGuardTests greps all client Swift source and fails on the token; CI job carries no real key secret and never logs env; key lives only in .env.local (gitignored) + edge env. |
| T-03-07 | Spoofing / Access Control | Unauthenticated call to /functions/v1/coaching | high | mitigate | Auth is enforced by the stock main router (VERIFY_JWT) ahead of dispatch; coaching function does not weaken it. Kong functions-v1 route applies cors; README documents that VERIFY_JWT must be on for real data (V2/V4). |

| T-03-01 | Tampering | Off-allowlist tag / banned content reaching the client | high | mitigate | index.ts applies validateCoachingResponse to provider output, retries once, else 502 — bad content never returned (tested). |
| T-03-SC | Tampering | npm/pip/cargo installs | high | mitigate | No package installs this phase (RESEARCH Package Legitimacy Audit: none).
