<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RevenueCat/App Store → app entitlement state | Premium/trial truth crosses from server-validated receipts into the app |
| device clock → daily-cap counter | User-controllable clock feeds the date-scoped free-tier counter |
| app-local storage → premium gate | A tampered local flag must NOT be able to grant premium |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-05-SPOOF | Tampering | EntitlementManager | high | mitigate | isPremium derived only from RevenueCat `customerInfo()` (server-validated); no separately-persisted premium bool that a jailbroken device could set |
| T-04-05-CAPBYPASS | Tampering | DailyCapTracker | low | accept | Clock rollback can reset the date-scoped free counter; no financial loss (only extra free usage); server-side counter out of scope for stateless-backend v1 — accepted risk, documented |
| T-04-05-TAGGATE | Tampering (spec) | CoachingResultModel / DailyCapTracker | high | mitigate | Cap gates generation frequency only; tags on existing suggestions stay visible at cap (MONE-01); no tag-hiding branch in cap/entitlement code |
| T-04-05-KEY | Information Disclosure | EntitlementManager | low | accept | RevenueCat public SDK key is safe-by-design to embed (unlike GEMINI_API_KEY); no secret-tripwire needed |
</threat_model>
