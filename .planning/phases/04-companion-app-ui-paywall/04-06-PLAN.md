<objective>
Close verification Gap 1 (Roadmap SC4 / MONE-01, MONE-02, MONE-03). PaywallView, DowngradeBanner, EntitlementManager, and DailyCapTracker are already built, individually correct, and unit-tested — but they have ZERO production call sites, so no real user can ever be capped, see the paywall, or see a downgrade. Output: BanterApp/Home/HomeModel.swift + HomeView.swift (new), a routed ContentView, and a completion hook on ValueDemoCoordinatorView.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RevenueCat/App Store → app entitlement state | Premium/trial truth crosses from server-validated receipts into the Home surface via EntitlementManager |
| device clock → daily-cap counter | User-controllable clock feeds the date-scoped free-tier counter the Home now constructs |
| app-local storage → premium gate / downgrade dedup | A tampered local flag must NOT grant premium; the downgrade marker is UI-only |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-06-SPOOF | Tampering | EntitlementManager wiring in HomeModel | high | mitigate | Home reads `isPremium` only from EntitlementManager (RevenueCat `customerInfo()`-derived); the wiring introduces no separately-persisted premium bool. The persisted last-known-status/downgrade marker is a UI-dedup flag only and never grants entitlement. |
| T-04-06-CAPBYPASS | Tampering | DailyCapTracker wiring | low | accept | Clock rollback can reset the date-scoped free counter (inherited T-04-05-CAPBYPASS); only extra free usage, no financial loss; unchanged by this wiring. |
| T-04-06-TAGGATE | Tampering (spec) | HomeView cap surface | high | mitigate | The cap surface (banner + paywall sheet) is presented OVER already-loaded cards; no tag-hiding branch is added and SuggestionCardView is untouched — tags stay visible at cap (MONE-01). |
| T-04-06-DOWNGRADE | Spoofing (UI) | DowngradeBanner dedup | low | accept | `lastSeenDowngrade` is a local UI-dedup marker; tampering only re-shows or suppresses a reassurance banner with no entitlement effect. |

No package-manager installs occur in this plan (RevenueCat was added and legitimacy-audited in Plan 01); no supply-chain checkpoint is required.
</threat_model>
