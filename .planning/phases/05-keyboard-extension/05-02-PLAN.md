<objective>
Rewrite the Phase 1 placeholder KeyboardViewController into a thin UIInputViewController that hosts a minimal SwiftUI suggestion list (KeyboardSuggestionsView) via a child UIHostingController, reads the cached suggestions from the App Group, inserts on tap, and always shows the globe key when required — the complete keyboard-side vertical slice, working with Full Access off and no network code path.

Purpose: This is the wedge. After this plan a user (on a device, manually) can switch to the Banter keyboard inside any chat app and tap a cached suggestion to insert it. It consumes exactly what Plan 01 writes.
Output: KeyboardSuggestionsView.swift (new SwiftUI view) + a rewritten KeyboardViewController.swift.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App Group container → BanterKeyboard process | The keyboard reads user-intimate suggestion text written by the app; runs inside OTHER apps' processes |
| BanterKeyboard process → host chat app text field | textDocumentProxy.insertText writes into whatever app is frontmost |
| BanterKeyboard target → network/SDK | KEYS-03: no network-capable code path may exist |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-04 | Elevation of Privilege | Future contributor flips RequestsOpenAccess to true or adds a network call to enrich suggestions in-keyboard | high | mitigate | KeyboardNetworkBoundaryGuardTests (Plan 01) fails CI on network tokens; RequestsOpenAccess stays false and is grep-asserted here; any flip is a KEYS-03 regression requiring human sign-off |
| T-05-05 | Information Disclosure | Stale intimate suggestion text lingering in the keyboard view after the conversation context changed | medium | mitigate | viewWillAppear re-reads the App Group so the keyboard reflects the latest single-key cache; the app overwrites (never appends) so no multi-conversation history is exposed |
| T-05-06 | Denial of Service | Oversized/malformed cached array jetsam-kills the extension (48MB ceiling) | low | mitigate | Render at most 3 rows (suggestions.prefix(3)), trivial view tree, no images beyond one SF Symbol, no third-party SDK linkage |
| T-05-SC | Tampering | npm/pip/cargo installs | high | mitigate | Not applicable — zero package installs this plan; BanterKeyboard's only dependency remains local BanterShared (verified in project.yml, unchanged) |
</threat_model>
